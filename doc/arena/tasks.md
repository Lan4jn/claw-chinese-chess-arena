## 注意事项

- 标记为 `*` 的任务可选，表示当前阶段只保留入口或占位，不要求完整实现。
- 任务状态以当前仓库真实状态为准，不再使用模板式全完成标记。
- 每个任务块都引用了对应需求，后续拆分 issue 或 PR 时应保留这种可追溯关系。
- 检查点任务用于增量验证，避免功能堆积到最后一次性联调。
- 属性测试用于验证跨输入空间始终成立的约束，不能被普通单元测试或一次性的集成测试替代。
